Boll Deluxe is, as it says on the tin, not very polished nor tested, and has notable jank associated with it.

There will probably be many other changes and fixes we forgot about that aren't mentioned here! Keep a keen eye out for them!

[Additions]
- Added total palettes skin option that decides how many palettes the skin has [Default: 13]
- Re-added Replays 
	- Replays are now saved as .bmov files, so they can be sent through social medias without data loss.
- Added a button in the main menu that redirects you to the Boll Team Youtube channel
- Skins can now hold skins for multiple characters in one skin folder by adding in two or more sets of sheets and merging the player.txts
	- lang [name] player.txt section is now accepted by the game as a replacement for lang for skin packs that use this behaviour
	- Do note that you cannot hold two skins of the same character in a folder with this method
	- The SMB1 Skins have been merged into 1 skin pack as an example of this.
	- 1.9.3 Sonic and 1.9.3 Super Sonic have been merged into 1 skin pack
- Added sheetname setting to player.txt that sets the name of the skin's sheet. (This can be used in combination with the above addition to have multiple skins using the same sheet)
- Added small shield offset skin setting (default: 4)

[Engine Technology]
- Level loading has been extremely sped up
- Added keepframebetween variable in charm that makes frames persist between sprites
- Added new charm sections #define match, #define meta and #define discard
	- meta runs between stages for saving and reading replay metadata. Check the META_TYPE variable for which actions to perform. 0 is saving, 1 is loading
	- discard runs when a movie is discarded. This should be used if your charm has unlockable data that needs to be reset after a replay ends.
	- match runs during game_start (the beginning of a run after the stage select). This only exists because i realised requiring charms to add gamemode start code 4 times for anything that should always happen is a bad idea.
	- To run code when a replay starts (like if your charm has unlockable data that needs to be swapped out), use this section and check for global.playback
- Codeblocks, the dummy objects, stones, and corks now all compile their code which should greatly optimize them
- The #define exitpipe section is now actually used
	- Doorexit sprites for characters now actually display
